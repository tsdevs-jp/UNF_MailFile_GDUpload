# Outlookアドイン「添付ファイル クラウド経由送信」取扱説明書

## 1. このアドインでできること

このアドインは Outlook でメール送信を行う際、通常の添付ファイルを自動で Google Drive にアップロードし、メール本文の先頭に共有用 URL を追記して送信するための社内向けツールです。

- 通常の添付ファイルを Google Drive にアップロードします
- メール本文の先頭に共有フォルダの URL を自動挿入します
- アップロード済みの添付ファイルは送信前にメールから取り外します
- 本文中に埋め込まれた画像などのインライン添付は、原則そのまま残します

## 2. 利用イメージ

送信時の流れは次のとおりです。

1. Outlook でメールを作成し、通常どおりファイルを添付します
2. 送信ボタンを押します
3. アドインが添付ファイルを Google Drive にアップロードします
4. メール本文の先頭に共有フォルダ URL を追記します
5. 元の添付ファイルをメールから外します
6. URL 付きメールを自動で送信します

本文の先頭には、おおむね次のような案内が追加されます。

```text
************************************************
本メールの添付ファイルは、
下記URLより確認・ダウンロードをお願いいたします。
https://drive.google.com/...
************************************************
```

## 3. 事前に必要なもの

利用前に、次の条件を満たしている必要があります。

- Windows PC
- Microsoft Outlook
- .NET Framework 4.8
- Visual Studio 2010 Tools for Office Runtime (VSTO Runtime 4.0)
- Google Drive を利用できる Google アカウント
- 管理者または配布担当者により本アドインがインストール済みであること

補足:

- Outlook アドインとして動作します
- インストーラーは管理者権限での導入を前提としています
- Google Drive の API クライアント情報はアドイン側に設定済みである前提です

## 4. 初回起動時の設定

アドイン起動時、次のどちらかが未設定の場合は設定画面が表示されます。

- Google Drive の認証が未実施
- 保存先の Google Drive ルートフォルダ ID が未設定

### 4-1. 設定画面の開き方

設定画面は次のタイミングで開けます。

- 初回起動時に自動表示される
- Outlook のメニューに追加される `Google Drive 設定` ボタンから開く

### 4-2. 認証手順

1. 設定画面で `認証` ボタンを押します
2. Google の OAuth 認証画面が表示されたら、利用する Google アカウントでサインインします
3. 認証成功メッセージが表示されたら完了です

認証情報は Windows のユーザーごとに保存されます。

### 4-3. ルートフォルダ ID の設定

`GoogleDriveRootFolderId` に、アップロード先の親フォルダ ID を設定して保存します。

Google Drive のフォルダ URL が次のような場合:

```text
https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStUvWxYz
```

フォルダ ID は次の部分です。

```text
1AbCdEfGhIjKlMnOpQrStUvWxYz
```

この ID を設定画面の入力欄に貼り付けて `保存` してください。

## 5. 普段の使い方

### 5-1. メール送信

1. Outlook で新規メールまたは返信メールを作成します
2. 必要なファイルを添付します
3. 通常どおり `送信` を押します
4. 進行状況ウィンドウが表示され、アップロード処理が進みます
5. 完了すると自動でメールが送信されます

### 5-2. Google Drive 側の保存内容

送信ごとに Google Drive 上へ専用フォルダが 1 つ作成されます。

- フォルダ名は `yyyyMMdd_HHmmss_件名` 形式です
- 件名が空の場合は `NoSubject` が使われます
- 件名に使えない文字は一部置換されます
- 長すぎる件名は短縮されます

### 5-3. 共有設定

作成されたフォルダには、リンクを知っている人が閲覧できる共有設定が付与されます。

注意:

- メール本文に記載された URL を受け取った相手は、URL からファイルを参照できます
- 機密性の高いファイルを扱う場合は、社内ルールに従って運用してください

## 6. このアドインが処理する対象

自動アップロードの対象になるのは、通常の添付ファイルです。

対象外になるもの:

- 本文内に埋め込まれた画像
- Outlook 上で通常添付ではない形式として扱われる添付
- MHTML 参照など、インライン扱いと判定されるもの

そのため、署名画像や本文装飾用画像は残り、通常添付したファイルだけが Google Drive に移されます。

## 7. エラー時の動き

Google Drive へのアップロードに失敗した場合、確認ダイアログが表示されます。

- `はい` を選ぶと、添付ファイルを付けたままそのまま送信します
- `いいえ` を選ぶと、送信を中止したままメール作成画面に戻ります

つまり、Google Drive 側で失敗しても、必要に応じて通常の添付メールとして送ることは可能です。

## 8. よくある確認ポイント

### 8-1. 設定画面が毎回出る

次を確認してください。

- Google アカウント認証が完了しているか
- ルートフォルダ ID が保存されているか
- 別の Windows ユーザーで Outlook を使っていないか

### 8-2. 送信時にアップロードが始まらない

次の可能性があります。

- 添付ファイルが付いていない
- 添付がすべてインライン扱いになっている
- Google Drive のクライアント設定が不足している
- Google 認証が切れている

### 8-3. Google Drive のどこに保存されたかわからない

設定済みのルートフォルダ配下に、送信ごとの新しいフォルダが作成されます。メール本文に追記された URL を開くと対象フォルダを直接確認できます。

## 9. ログの確認方法

設定画面の `ログを開く` から診断ログを開けます。

ログファイルの保存先:

```text
C:\Users\<Windowsユーザー名>\AppData\Roaming\UNF_MailFile_GDUpload\diagnostic.log
```

認証トークンの保存先:

```text
C:\Users\<Windowsユーザー名>\AppData\Roaming\UNF_MailFile_GDUpload\GoogleDriveToken
```

## 10. 補足事項

- 一時ファイルは送信処理中にローカルの一時フォルダへ展開され、処理後に削除されます
- 同じメールで処理中に重複送信しようとすると、処理中メッセージが表示されます
- アップロード完了後の再送時に二重処理しないよう、内部フラグで制御されています

## 11. 利用者向けの簡易手順

迷ったときは次の流れで使ってください。

1. Outlook でメールを作る
2. ファイルを添付する
3. `送信` を押す
4. 初回だけ Google 認証と保存先フォルダ ID を設定する
5. 本文に追加された Google Drive の URL が相手に届くことを確認する

以上で、通常の添付送信を Google Drive 経由の共有送信に置き換えて利用できます。
