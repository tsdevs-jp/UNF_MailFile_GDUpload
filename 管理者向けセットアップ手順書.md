# Outlookアドイン「添付ファイル クラウド経由送信」管理者向けセットアップ手順書

## 1. 目的

この文書は、本アドインを配布・導入・初期確認する管理者向けの手順書です。

対象:

- 社内PCへの導入担当者
- 配布パッケージ作成担当者
- 動作確認や障害切り分けを行う担当者

## 2. アドイン概要

本アドインは Outlook 用の VSTO アドインです。メール送信時に通常添付ファイルを Google Drive にアップロードし、本文先頭へ共有 URL を挿入して送信します。

主な動作:

- Outlook の `ItemSend` イベントで処理開始
- 通常添付のみを抽出
- 一時フォルダへ添付を保存
- Google Drive 上に送信単位の共有フォルダを作成
- ファイルをアップロード
- 本文先頭にフォルダ URL を挿入
- 元添付を削除して再送信

## 3. 動作要件

導入先 PC は次の前提を満たしている必要があります。

- Windows
- Microsoft Outlook
- .NET Framework 4.8
- Visual Studio 2010 Tools for Office Runtime 4.0
- Google アカウントでの OAuth 認証が可能であること

補足:

- インストーラーは管理者権限での実行を前提としています
- Outlook アドイン登録はユーザー領域 `HKCU` に作成されます
- インストール先は既定で `C:\Program Files\Unfake\UNF_MailFile_GDUpload\` です

## 4. 構成上の重要ポイント

### クライアント設定

[App.config](C:/Development/UNF_MailFile_GDUpload/App.config) の `appSettings` に次の項目が必要です。

- `GoogleDriveApplicationName`
- `GoogleDriveClientId`
- `GoogleDriveClientSecret`

これらが未設定だと、アドイン起動時または送信時に設定画面表示や警告が発生し、アップロード処理は実行されません。

### 利用者ごとの設定

ユーザー設定として保存される項目:

- `GoogleDriveRootFolderId`

これは利用者ごとに保持されるため、PC 全体のインストールとは別に各ユーザーの初回設定が必要です。

### 認証トークン保存先

Google OAuth のトークンは次に保存されます。

```text
C:\Users\<Windowsユーザー名>\AppData\Roaming\UNF_MailFile_GDUpload\GoogleDriveToken
```

このフォルダには、原則として現在の Windows ユーザーのみアクセスできるよう ACL 設定が試みられます。

## 5. 導入手順

### 手順A: 利用者PCへインストーラーで導入する場合

1. 対象 PC に .NET Framework 4.8 と VSTO Runtime 4.0 があることを確認する
2. 配布済みセットアップ EXE を管理者権限で実行する
3. インストール完了後、Outlook を再起動する
4. Outlook 上でアドインが読み込まれていることを確認する
5. 初回起動時に利用者へ Google 認証とルートフォルダ ID 設定を案内する

### 手順B: 開発環境からセットアップを作成して配布する場合

1. Visual Studio で Release ビルドを実行する
2. `bin\Release\` に生成物が揃っていることを確認する
3. [Setup\setup.iss](C:/Development/UNF_MailFile_GDUpload/Setup/setup.iss) を Inno Setup でコンパイルする
4. `Setup\output\UNF_MailFile_GDUpload_Setup_1.0.0.exe` を配布物として使用する

関連資料:

- [Setup\README_ビルド手順.md](C:/Development/UNF_MailFile_GDUpload/Setup/README_ビルド手順.md)
- [Setup\setup.iss](C:/Development/UNF_MailFile_GDUpload/Setup/setup.iss)

## 6. インストーラーの実施内容

Inno Setup ベースのインストーラーは、主に次を実施します。

- `bin\Release\` の成果物をアプリフォルダへコピー
- Outlook アドイン登録を `HKCU\Software\Microsoft\Office\Outlook\Addins\UNF_MailFile_GDUpload` に作成
- VSTO マニフェストパスをレジストリに登録
- 証明書を `TrustedPublisher` と `Root` ストアに登録
- アンインストール時に証明書削除を試行

## 7. 導入後の初期確認

導入直後は次の観点で確認してください。

### Outlook 起動確認

- Outlook 起動時にアドインエラーが出ない
- メニュー上に `Google Drive 設定` ボタンが表示される

### 設定画面確認

- 設定画面が開ける
- クライアント設定ステータスが正常表示される
- `認証` ボタンから Google 認証に進める

### 送信確認

1. テストメールを作成する
2. 任意のファイルを 1 つ添付する
3. `送信` を押す
4. 進行状況ウィンドウが表示される
5. Google Drive に共有フォルダが作成される
6. メール本文先頭に URL が入る
7. 元添付が外れた状態で送信される

## 8. 利用者へ案内すべき内容

配布時には少なくとも次を利用者へ伝えると運用が安定します。

- 初回のみ Google 認証が必要
- 保存先の Google Drive フォルダ ID が必要
- 失敗時は添付付きの通常送信へ切り替え可能
- メール本文の共有 URL を受信者が使う運用になる

## 9. 障害切り分け

### ログ確認

診断ログ:

```text
C:\Users\<Windowsユーザー名>\AppData\Roaming\UNF_MailFile_GDUpload\diagnostic.log
```

まず確認したい観点:

- 起動時にクライアント設定不足が出ていないか
- 添付抽出時に例外が起きていないか
- Google Drive アップロード時に例外が出ていないか

### よくある原因

- `GoogleDriveClientId` または `GoogleDriveClientSecret` 未設定
- `GoogleDriveRootFolderId` 未設定
- Google 認証未実施または期限切れ
- 導入先に .NET Framework 4.8 / VSTO Runtime がない
- Outlook 側でアドインが無効化されている

## 10. 運用上の注意

- 共有フォルダ権限は `anyone` / `reader` で作成されます
- URL を知っている人はフォルダ内ファイルを参照できます
- 機密情報の取り扱いは組織ルールに合わせてください
- `App.config` に含まれる Google API の設定値は厳重に管理してください
